 Q: Suka programming ? ,jika ya bahasa apakah yang sering digunakan ?
 Y: C++

 Q: Jenis aplikasi apa yang sering kamu buat?, mengapa ?
 Y: Game, dan aplikasi-aplikasi yang 'just for fun', karena simpel dan ada 
    kesenangan sendiri.

 Q: Echo staff pernah melihat beberapa sample game kamu. Bisa kamu ceritakan 
    garis besar teknis pembuatan game-game itu?
 Y: Pembuatannya dari ide kasar aja trus dibuat gambar-gambarnya.
    Setelah kebayang gameplaynya, baru aku coding dan sesekali test run untuk 
    menguji gambarnya keluar atau tidak. Kalau gambarnya berhasil diblit, 
    baru coding gameplaynya, lalu terakhir menu gamenya (jarang kubikin menu 
    game, karena sebagian besar alasanku bikin game cuman buat kesenangan 
    sendiri aja). Hehe lagian game-gamenya setengah jadi semua kan :P

 Q: Game komersial di Linux, menurut kamu bagaimana prospeknya?
 Y: Tergantung dari pengguna Linux itu sendiri.
    Sampai sekarang salah satu yang menghambat orang-orang (biasanya windows 
    user) pindah ke Linux itu keterbatasan game. Ini juga karena banyak 
    industri game komersial hanya merilis game-nya yang kompatibel untuk \
     Windows (dan juga Mac OS). Jika pengguna linux yang merupakan gamers 
    semakin banyak, para developer game pun mau gak mau harus merilis / 
    porting game-gamenya untuk Linux.

 Q: Apakah game programming menurut kamu adalah juga bentuk hacking? dan 
    menurut kamu apa ciri-ciri khas pemrograman game dibanding pemrograman 
    di bidang lain?
 Y: Mungkin bisa dibilang iya, tapi dalam bentuk pasifnya bukan aktifnya.
    Misalkan dalam pemrograman game, kita juga harus melihat 'aman' nya game 
    tersebut. Seperti penggunaan memori yang tidak berlebihan, atau mungkin 
    menjaga tidak ada kecurangan atau flaw dalam game tersebut yang bisa 
    dihack oleh player (terutama game online). Mungkin ciri khas dalam 
    pemrograman game terletak di game loop dan artificial intelligence-nya.

 Q: SDL biasanya menjadi pilihan banyak programmer game di Linux. Apakah 
    library yang sama juga merupakan pilihan kamu? atau kamu ada pilihan lain?
 Y: Iya, sampai sekarang menurutku yang paling 'enak' dipake itu SDL. 
    Simpel tapi powerful.

 Q: Pilih mana, DirectX atau OpenGL? Dan apa alasannya?
 Y: OpenGL dong. Alasan utama karena OpenGL itu bersifat terbuka, 
    cross-platform, dan lebih fleksibel.

 Q: Lebih OK mana, game windows jalan diatas WINE atau game native Linux?
 Y: Tentu game native linux dong. Walaupun sekarang memang banyak game-game 
    Windows yang berhasil dijalanin menggunakan WINE, tapi masih banyak 
    masalah-masalahnya. Mungkin kalau directX (dan API atau library lain 
    di windows yang diperlukan untuk menjalankan game) bisa 100% diemulate, 
    WINE merupakan pilihan untuk gamer windows yang migrasi ke linux.

 Q: Ok,cukup tentang masalah game, sebenarnya cita - cita terbesar kamu yang

 Y: Menjadi manusia yang berguna untuk orang banyak
